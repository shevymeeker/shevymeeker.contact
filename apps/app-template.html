How it stays offline-first

Each app ships with an app-shell service worker (cache-first for UI), IndexedDB via Dexie.js for user data, and a manifest for installability.

  • App shell cached on first load so navigation works offline.
  • Dexie.js handles data; add tables per entity (clients, jobs, tasks).
  • Service worker refreshes assets in the background after activation.
  • Optional PDF exports via jsPDF/react-pdf for quotes or receipts.
  • Ships on GitHub Pages with a manifest for install prompts.
